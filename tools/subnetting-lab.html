// Layer 3 — IP Addressing

SUBNET
TING

Every IP address is made of 32 bits split into a network part and a host part. The subnet mask decides where that split happens. Enter any IP/CIDR below to see it broken down to the bit level.

NET
// What it is
Network Bits
The left portion of the address. All devices sharing these bits are on the same network. Represented by the 1s in the subnet mask.
HOST
// What it is
Host Bits
The right portion. These bits identify individual devices within a network. Represented by the 0s in the subnet mask.
CIDR
// What it is
CIDR Notation
The /24 after an IP address. It's simply a count of how many bits are network bits. /24 means 24 network bits, 8 host bits.
// Why Does Subnetting Exist?

Imagine your office building has 1,000 computers all on the same network. Every time any device sends a broadcast — an ARP request, a DHCP discover — all 1,000 machines receive and process it. That's a massive amount of wasted traffic called a broadcast storm.

Subnetting solves this by dividing one large network into smaller logical groups. Each group (subnet) only sees broadcasts from its own members. You also get better security — the Accounting subnet can be isolated from the Guest WiFi subnet — and easier troubleshooting, since problems stay contained.

At its core, subnetting is just deciding how to split 32 bits between identifying the network and identifying the individual host. The subnet mask is the tool that defines that split.

// The Street Address Analogy
🏘 STREET = NETWORK

Think of the network portion like the street name — "Oak Street". Every house on Oak Street shares this identifier. If you're on Oak Street and want to talk to another house on Oak Street, you walk over directly.

192.168.1.__ ← network part (Oak Street)
🏠 HOUSE NUMBER = HOST

The host portion is like the house number — "42". It uniquely identifies one specific device within that network. Two houses on different streets can share the same number — just like two subnets can have a host .1.

__.__.__.42 ← host part (house number)
Key insight — To send a letter to a different street, it must go through the post office (router). Same in networking — traffic between subnets always passes through a router. Traffic within the same subnet is delivered directly by the switch.
// How the Subnet Mask Works

The subnet mask is a 32-bit number that acts like a stencil placed over your IP address. Where the mask has a 1, that bit belongs to the network. Where it has a 0, that bit belongs to the host. Your device performs a bitwise AND operation between the IP and the mask to find the network address.

IP ADDRESS
192.168.1.100
11000000.10101000.00000001.01100100
SUBNET MASK (/24)
255.255.255.0
11111111.11111111.11111111.00000000
RESULT (AND) = NETWORK
192.168.1.0
11000000.10101000.00000001.00000000

The last octet of the mask is all 0s, so the last octet of the result is also all 0s — that's 192.168.1.0, the network address. The host part (.100) gets zeroed out, leaving just the network identity.

// Special Addresses in Every Subnet
Network Address

The first address in the subnet. All host bits are 0. This address identifies the subnet itself — it cannot be assigned to any device.

192.168.1.0/24 → Network
Usable Hosts

Every address between the network and broadcast addresses. These are assigned to devices — computers, printers, routers, phones.

192.168.1.1–254 → Hosts
Broadcast Address

The last address in the subnet. All host bits are 1. A packet sent here is delivered to every device on the subnet simultaneously.

192.168.1.255/24 → Broadcast
// The Formula You Need to Memorize
USABLE HOSTS
2h − 2

Where h = number of host bits (32 minus the prefix). The −2 removes the network address and broadcast address. A /24 has 8 host bits → 2⁸ − 2 = 254 hosts.

NUMBER OF SUBNETS
2s

Where s = number of bits borrowed from the host portion. Borrowing 3 bits from a /24 gives you /27s → 2³ = 8 subnets, each with 30 hosts.

// How to Subnet — Step by Step
STEP 1
Count the host bits
Host bits = 32 − CIDR prefix. For /24: 32−24 = 8 host bits.
STEP 2
Calculate total addresses
Total = 2^(host bits). For 8 bits: 2^8 = 256 addresses.
STEP 3
Subtract 2 for usable hosts
Subtract network address + broadcast. 256−2 = 254 usable hosts.
STEP 4
Find network & broadcast
Network = first address. Broadcast = last address (all host bits = 1).
