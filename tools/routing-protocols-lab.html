BGP & EIGRP
// Exterior Gateway Protocol

BORDER
GATEWAY
PROTOCOL

BGP is the routing protocol of the internet. It connects Autonomous Systems — entire networks owned by different organizations — and controls how traffic flows between them globally.

// What is BGP?

An Autonomous System (AS) is a collection of IP networks managed by a single organization — like Google (AS15169), Comcast (AS7922), or a university. BGP is how these organizations advertise their IP prefixes to the world and learn routes from each other.

Unlike interior protocols (OSPF, EIGRP) that optimize for speed, BGP optimizes for policy and business relationships. An ISP can prefer a partner's path over a competitor's even if the competitor is technically faster. BGP enables this through ranked path attributes.

BGP sessions run over TCP port 179 and must be manually configured. Two BGP routers with an active session are called peers. BGP never auto-discovers neighbors — every peering is intentional.

// Protocol Type
Path Vector
BGP records the full sequence of AS numbers a route traverses (AS_PATH). If your own AS appears in an incoming path, you reject it — that's loop prevention.
// Scale
Internet-Wide
The global BGP table carries over 950,000 IPv4 routes. BGP has been routing the internet since 1994 and is the only protocol built for this scale.
// Session Types
eBGP / iBGP
eBGP connects different ASes. iBGP distributes BGP routes within the same AS. iBGP peers don't re-advertise to each other without route reflectors.
// Path Attributes — Decision Order

BGP evaluates attributes in strict order. The first attribute that differs between two routes decides the winner. Memorize this for your interview.

// 1st
Weight
Cisco-only. Higher = better. Never advertised. Local to one router only.
// 2nd
Local Pref
Higher = better. Shared within AS. Controls which exit point traffic uses.
// 3rd
AS_PATH
Shorter = better. Fewer AS hops = more direct. Also prevents loops.
// 4th
Origin
IGP > EGP > Incomplete. How the route entered BGP.
// 5th
MED
Lower = better. Hints to external peers which path to use entering your AS.
// Mnemonic
We Love Oranges AS Oranges Mean Pure Refreshment
Weight, Local-Pref, Originate, AS_PATH, Origin, MED, Paths, Router-ID

// Interior Gateway Protocol

EIGRP

Enhanced Interior Gateway Routing Protocol — a hybrid protocol combining distance-vector simplicity with link-state intelligence. EIGRP uses the DUAL algorithm to guarantee loop-free, instantly-converging paths inside your network.

// How EIGRP Works

EIGRP discovers neighbors automatically by sending Hello packets to multicast 224.0.0.10 every 5 seconds. Any EIGRP router on the same segment that hears your Hello becomes your neighbor — zero manual config, unlike BGP.

Once neighbors form, routers exchange full topology tables using UPDATE messages. After that, only partial triggered updates are sent — only when something changes, only to affected routers. Very bandwidth-efficient.

The core is DUAL (Diffusing Update Algorithm). DUAL computes the best path (Successor) and a pre-validated backup (Feasible Successor). If the primary fails, EIGRP switches to backup in milliseconds — zero recalculation needed.

// Algorithm
DUAL
Diffusing Update Algorithm. Guarantees loop-free paths at every moment — even while topology changes are still propagating across the network.
// Metric
Composite
Default: 256 x (BW + Delay). Bandwidth = minimum along path. Delay = sum of all delays. Lower metric = better path.
// Convergence
Sub-second
With a Feasible Successor precomputed, failover is instant — no query process. Even without one, DUAL converges faster than most IGPs.
// Key Terms
// Term 01
Feasible Distance (FD)
Total metric cost from THIS router to the destination. The lowest FD route becomes the Successor — installed in the routing table as the primary path.
// Term 02
Reported Distance (RD)
The metric a neighbor reports for a destination — what it thinks the cost is from itself. Also called Advertised Distance (AD). Used in the Feasibility Condition check.
// Term 03
Successor
The primary next-hop — the neighbor offering the lowest FD. Installed in routing table. Multiple successors allowed for load balancing across equal-cost paths.
// Term 04
Feasible Successor
A backup path satisfying the Feasibility Condition: its RD must be strictly less than current FD. Mathematically loop-free. Stored in topology table for instant failover.

Feasibility Condition: A route qualifies as Feasible Successor only if neighbor's RD < this router's current FD. This guarantees the backup cannot loop through you.

Example: Your FD to 10.1.0.0/24 = 28160. Neighbor X reports RD = 25600. Since 25600 < 28160, X qualifies as Feasible Successor. ✓

Use BGP When...

✦ Connecting to one or more ISPs (internet-facing)

✦ Your org has its own public AS number and IP block

✦ You need traffic engineering — controlling ingress/egress paths

✦ Running multi-homed connections for redundancy

✦ Building large-scale MPLS VPN backbones

Use EIGRP When...

✦ Running a Cisco-heavy enterprise network

✦ Fast convergence is critical — voice, video, real-time

✦ You need unequal-cost load balancing across WAN links

✦ Simpler config than OSPF is preferred

✦ WAN links have varying speeds needing smart metric routing
